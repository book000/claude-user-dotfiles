# PR 本文と実装の整合性検証

PR が作成または更新されました。以下の手順で PR 本文と実装の整合性を検証してください。

## 検証手順

1. `git diff origin/master...HEAD` で実際のコード変更を確認
2. PR 本文に記載された以下の項目が実装と一致しているか確認:
   - デフォルト値（環境変数のデフォルト値など）
   - 環境変数名
   - セレクター
   - タイムアウト値
   - ファイルパス
   - メソッド名・関数名
3. 不一致がある場合は PR 本文を更新

## 出力

検証結果を簡潔に報告してください:
- 一致している場合: 「PR 本文と実装が一致しています」
- 不一致がある場合: 不一致箇所を指摘し、PR 本文を更新
